### 1. URL ซ้ำตอน Crawl
**ไฟล์:** `Backend/src/web_url_tester.py`  
**บรรทัด:** 182–234

**ปัญหา:**
- `https://v89tech.com` กับ `https://v89tech.com/` ถูกมองว่าเป็น 2 หน้าต่างกัน
- ใช้ `visited` set เดียว พอ add link เข้า queue ก็ใส่ visited ด้วย พอจะ crawl จริงก็โดน skip ทุกหน้า ได้แค่ home หน้าเดียว

**วิธีแก้:**
```python
# เพิ่มฟังก์ชัน normalize URL
def _norm_url(u: str) -> str:
    return u.rstrip("/").split("?")[0].split("#")[0]

# แยก set เป็น 2 ตัว
queued: set = {_norm_url(url)}   # กัน URL เข้า queue ซ้ำ
crawled: set = set()             # กัน snapshot ซ้ำ + redirect

# เช็ค URL จริงหลัง redirect
final_url = page.url
norm_final = _norm_url(final_url)
if norm_final in crawled:
    continue
crawled.add(norm_final)
```

### 2. Screenshot ซ้ำ (หน้าเดิมถ่ายหลายรูป)
**ไฟล์:** `Backend/src/web_url_tester.py`  
**บรรทัด:** 454–546

**ปัญหา:**
- AI สร้าง test cases หลายตัวที่ goto หน้าเดิม เช่น home 3 ตัว, our-services 5 ตัว → screenshot ซ้ำ
- หน้าที่ AI ไม่ได้ generate test ให้ (เช่น ai-monitoring, workshop) ไม่มี screenshot เลย

**วิธีแก้ — แบ่งเป็น 2 Phase:**

**Phase 1:** รัน AI tests ก่อน ถ่าย screenshot เฉพาะครั้งแรกที่เจอ URL นั้น
```python
snapped_pages: set = set()

final_norm = _norm_url(page.url)
if final_norm not in snapped_pages:
    shot = capture.capture_playwright(...)   # ถ่ายครั้งแรก
    snapped_pages.add(final_norm)
else:
    shot = None                              # หน้าซ้ำ ไม่ถ่าย
```

**Phase 2:** วน crawled pages ทุกหน้า ถ้ายังไม่มี screenshot → ถ่าย SNAP_
```python
for idx, ctx in enumerate(page_contexts, start=1):
    page_url = ctx.get("url", url)
    if _norm_url(page_url) in snapped_pages:
        continue   # ถ่ายไปแล้ว ข้าม
    # ... ถ่าย screenshot หน้าที่ขาด
```

**ผลลัพธ์:** ทุกหน้าได้ screenshot ครบ ไม่มีซ้ำ

### 3. เว็บช้า/Timeout ไม่ Crash
**ไฟล์:** `Backend/src/web_url_tester.py`  
**บรรทัด:** 117–135

**ปัญหา:**
- เว็บที่โหลดช้า timeout แล้วโปรแกรม crash ทั้งหมด มี traceback ยาว

**วิธีแก้:**
```python
try:
    page.goto(url, wait_until="networkidle", ...)    # ลองก่อน
except Exception:
    try:
        page.goto(url, wait_until="domcontentloaded", ...)   # fallback
    except Exception as e:
        return {"url": url, "error": str(e), ...}    # ไม่ crash
```

### 4. Screenshot Timeout เพราะ Animation
**ไฟล์:** `Backend/src/screenshot_manager_capture.py`  
**บรรทัด:** 160–164

**ปัญหา:**
- เว็บที่มี CSS animation วิ่งอยู่ตลอด Playwright รอ animation จบแล้ว timeout 30วิ

**วิธีแก้:**
```python
page.screenshot(
    path=str(file_path),
    full_page=full_page,
    type=self.format,
    animations="disabled",   # หยุด animation ก่อนถ่าย
```

### 5. อ่าน Model จาก .env ได้จริง
**ไฟล์:** `Backend/src_config.py`  
**บรรทัด:** 11–13 และ 27–28

**ปัญหา:**
- `load_dotenv()` ไม่รู้ว่า `.env` อยู่ที่ `Backend/` ทำให้อ่าน `AI_MODEL` จาก env ไม่ได้
- `AI_MODEL` hardcode เป็น `"gemini-2.5-flash"` ในตัว class ไม่ได้อ่านจาก env

**วิธีแก้:**
```python
# ชี้ path .env ให้ตรง
_env_path = Path(__file__).parent / ".env"
load_dotenv(dotenv_path=_env_path)

# อ่านจาก env
AI_MODEL = os.getenv("AI_MODEL", "gemini-2.5-flash")
```

## ปัญหาที่พบระหว่างวัน

| ปัญหา | สาเหตุ | สถานะ |
|---|---|---|
| Gemini 429 quota หมด | ใช้ gemini-2.5-flash ฟรี 20 req/วัน รัน debug หลายรอบ | reset พรุ่งนี้ 07:00 น. |

| gemini-2.0-flash limit 0 | project นี้ไม่มี quota สำหรับ model นี้ | เปลี่ยนกลับ 2.5-flash |
